🎬 How It Actually Executes

The Simple Version

Every day at 7 AM NST, n8n automatically runs your workflow. It:

  1. Wakes up at exact time (7:00:00 AM)
  2. Runs through 9 nodes sequentially
  3. Each node processes data & passes to next
  4. OpenAI generates unique post (2 seconds)
  5. Posts to Facebook (1 second)
  6. Logs to Google Sheets
  7. Workflow finishes (total: ~3 seconds)
  8. Post appears on your page ✅

🔧 Node-by-Node Execution

Node 1: Schedule Trigger

⏱️ Schedule Trigger (Cron)

When: Every day at 7:00 AM NST

Cron: 15 1 * * * (7 AM NST = 1:15 AM UTC)

Output:
Signals next node to start
Passes: { timestamp, executionId }

Node 2: Get NST Context

📅 Get NST Context (Function)

What it does: Gets current date/time in Nepal timezone

Input: Timestamp from Node 1
Output:
{
  timestamp: "2026-06-14T01:15:00Z",
  dayName: "Monday",
  dateEng: "14/06/2026",
  nepaliMonth: "Baishakh",
  dayOfWeek: 1,
  hour: 7,
  isWeekend: false
}

Time Taken: 10ms

Node 3: Select Content Category

🎲 Select Content Category (Function)

What it does: Randomly picks content type based on day of week

Input: dayOfWeek from Node 2

Logic:

if Monday {
  Pick from: Study Tips (40%), Good Morning (60%)
}
if Wednesday {
  Pick from: Announcement (60%), Good Morning (40%)
}
if Friday {
  Pick from: Weekly Wrap (40%), Motivation (60%)
}
// ... etc for other days
Output:
{
  category: "study_tips",
  dayOfWeek: 1
}

Time Taken: 10ms

Node 4: OpenAI Content Generation ⚡ MAIN NODE

🤖 OpenAI API (Chat Completion)

What it does: Calls OpenAI to generate unique post content

Input:
- dayName: "Monday"
- category: "study_tips"
- Plus system prompt (detailed instructions)

API Call:

POST https://api.openai.com/v1/messages

{
  "model": "gpt-4-turbo",
  "temperature": 0.8,
  "max_tokens": 300,
  "messages": [
    {
      "role": "system",
      "content": "You are a content creator for KMC..."
    },
    {
      "role": "user",
      "content": "Generate study tips post for Monday..."
    }
  ]
}
Output (Example):
{
  "text": "📚 STUDY TIP: Active Recall...",
  "hashtags": "#StudyTips #KMC #UCDL",
  "category": "study_tips"
}

Time Taken: 1.9 seconds (API latency)

⚠️ This is the slowest node (waiting for OpenAI)

Node 5: Parse AI Response

📝 Parse Response (Function)

What it does: Extracts text from OpenAI's JSON response

Input: Raw OpenAI response (JSON)

Processing:

1. Extract text field
2. Extract hashtags field
3. Combine into finalPost
4. Count characters
5. Validate length (must be 180-250)
Output:
{
  postText: "📚 STUDY TIP: Active Recall...",
  hashtags: "#StudyTips #KMC #UCDL",
  finalPost: "[Combined text + hashtags]",
  characterCount: 215
}

Time Taken: 10ms

Node 6: Post to Facebook ⚡ CRITICAL NODE

📱 Facebook Graph API (HTTP POST)

What it does: Posts content to your Facebook page

Input:
- finalPost (text + emojis + hashtags)
- Access Token
- Page ID

API Call:

POST https://graph.facebook.com/v18.0/123456789/feed

{
  "message": "📚 STUDY TIP: Active Recall helps memory!...",
  "access_token": "EABC...",
  "published": true
}
Output (Success):
{
  id: "123456789_987654321",
  created_time: "2026-06-14T01:15:03+0000"
}

Time Taken: 900ms (API latency)

✅ Post is NOW LIVE on Facebook page!

Node 7: Prepare Log Data

📋 Prepare Log Data (Function)

What it does: Formats all data for Google Sheets logging

Input: All data from previous nodes

Processing: Collect and format for sheet insertion

Output (Row for Google Sheets):
{
  date: "14/06/2026",
  dayName: "Monday",
  category: "study_tips",
  postText: "📚 STUDY TIP...",
  postId: "123456_987654",
  characterCount: 215,
  timestamp: "01:15:03 NST",
  status: "SUCCESS"
}

Time Taken: 10ms

Node 8: Log to Google Sheets

📊 Google Sheets API (Append Row)

What it does: Saves log data to Google Sheets for tracking

Input: Formatted log data from Node 7

API Call:

POST sheets.googleapis.com/v4/spreadsheets/[ID]/values/append

{
  "values": [
    ["14/06/2026", "Monday", "study_tips", "📚 STUDY...", 
     "123456_987654", 215, "01:15:03", "SUCCESS"]
  ]
}
Output:
{
  spreadsheetId: "abc123xyz",
  updatedCells: 8,
  updatedRows: 1
}

Time Taken: 110ms

✅ Data now saved in Google Sheet for your tracking!

Node 9: Complete

✅ Execution Complete (No Operation)

What it does: Marks workflow as successfully finished

Final Status: SUCCESS ✅

⚠️ What Happens If Something Goes Wrong

Node 4 (OpenAI) Fails

❌ Without Error Handling

OpenAI API times out

↓

Entire workflow fails

↓

No post published

↓

You don't know why

Result: Silent failure

✅ With Error Handling

OpenAI API times out (1st try)

↓

n8n auto-retries (2nd try)

↓

If still fails → Email alert

↓

Google Sheet logs: FAILED

↓

You get notification

Result: You know & can fix

Node 6 (Facebook) Fails

Scenario: Facebook token expired

What happens:
  1. n8n tries to POST to Facebook
  2. Gets 401 error (Unauthorized)
  3. Auto-retry triggers (tries again)
  4. Still fails
  5. Stops execution at Node 6
  6. Google Sheets gets: status = "FAILED"
  7. Email alert sent to you
  8. Message: "Facebook token invalid"
You then: Generate new token, update in n8n, next day posts successfully

Node 8 (Google Sheets) Fails

Scenario: Google Sheets API rate limit hit

What happens:
  1. Sheets API returns rate limit error
  2. Node 8 fails
  3. But workflow is set to "continue on error"
  4. Post already published on Facebook ✓
  5. Sheets logging just skipped
  6. Workflow completes: status = "PARTIAL_SUCCESS"
Impact: Post is live, but not logged. Minor issue, easily recoverable.

Multiple Failures

Rare case: Both OpenAI AND Facebook fail

7:00:00 - Trigger fires
7:00:00.2 - Get context ✓
7:00:00.3 - Select category ✓
7:00:00.31 - Call OpenAI... ERROR! ❌
          - Auto-retry... ERROR! ❌
          - Workflow stops

Result:
- No post published
- Email alert sent
- Status: FAILED
- Google Sheet: No entry (workflow didn't reach Node 7)

Recovery:
- Check OpenAI API status
- Check OpenAI credits
- Fix issue
- Workflow runs again tomorrow

Bottleneck Analysis

⏱️ The Slowest Part: Waiting for OpenAI API (1.9 seconds)

This is why:

  • OpenAI's servers receive millions of requests
  • GPT-4 takes time to generate quality content
  • Network latency (request travels to US servers)

Can we make it faster?

  • Use GPT-4o instead of GPT-4-turbo (slightly faster)
  • Reduce max_tokens (trades quality for speed)
  • Use a faster model (GPT-3.5, but lower quality)

Is 3.26 seconds acceptable?

✅ YES! Post is live before users even know it ran!
